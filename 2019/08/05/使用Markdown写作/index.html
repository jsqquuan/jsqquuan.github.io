使用Markdown写作

Markdown是一种轻量级标记语言，创始人为约翰·格鲁伯（英语：John Gruber）。它允许人们
“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML（或者HTML）文档”。这种语言吸
收了很多在电子邮件中已有的纯文本标记的特性。 –wiki百科

  Markdown这种语法，应该是语言，在以前就使用过了，不过那时候用的比较简单，工具是用的网易云笔记，主要作用呢，就是报告一下进度，以及目前的问题之类的，但是那个时候，无非就是天天写流水一般的文章，虽然相比其他人来说，我的流水算是没那么单纯，但是还是只用到了几个字符而已，”#”控制标题大小，”···”后跟随代码……等等一系列，简单，易用，写出来的东西耶尚可，就是有一点，感觉编辑起来整个文章和用空格键去代替word中的首行缩进的感觉差不多，那种怎么说呢，看似没事，换个工具迟早出问题。

  为了贯彻写博客这件事，我这两天读了几篇关于Markdown的文章，对于这种语法的存在，如果是一个日常经常使用Word做报告或者写作的人来说，这就是一种没什么必要，且多余的东西，想想 用了大半年去学习一系列的办公软件（对于非it从业者来说，或是一种常见的现象），基本上日常都是使用这一工具去工作，特别是整个公司都是使用同一Word版本，或者需要打印的，其实再学习Markdown，无疑是累赘而且多余的。但是反观我这一系列码农，每天学习着不同的框架，踩着不一样的坑，再多学一种语法，对日常丝毫没有影响，或许哪一天衍生出一种用Markdown标记的编程语言也说不定，而且这种语法，看着跟注释一样的标记，实在让我觉得舒服，不用转换也能一眼看出各处的意思，好比在看一堆堆老代码的时候，打开了一篇写满注释的代码，久旱逢甘霖一般。

  好吧，我这两段话对于使用Markdown毫无帮助，都是废话，说实在我现在也不适合说什么帮助一类的东西，我也在学习，但是这里推荐一下sainho这位作者发布在少数派上的两篇文章：《Markdown 完全入门(上篇/下篇)》文章通篇对于Markdown的语法特性，以及场景，和Word对比概述的非常完整，虽然对于入手帮助不大，但是基于搞懂Markdown这个问题，基本上是很完善了，我非常赞同作者的一个观点，类似于Markdown这样的一个语言来说，通过一篇文章去读懂，比如经典的《XXX从入门到___》基本上是不可能的，首先书籍就是千页起步，一篇文章下来不超过1W字，对于一名读者来说，面对屏幕过长时间的阅读会产生疲劳，更多的人也在读书的过程中逐渐放弃，所以更好的是去了解这一语言，知道他的历史，以及搞明白什么时候使用到它，然后知道怎么去找到使用方法，就是最佳的学习，剩下的，就是在使用期间，通过学到的方法需寻找解决方法（其实我认为就是关键字，有时候寻找篇文章或者寻找一个东西，最大的问题就是关键字），这样就可以完美的解决问题，把坑一个个踏过，如果上面作者所说的，Markdown依赖的不是视觉化的工具，而是肌肉记忆，最后实现编辑内容和排版一气呵成。
