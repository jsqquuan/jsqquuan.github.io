使用Markdown写作

Markdown是一种轻量级标记语言，创始人为约翰·格鲁伯（英语：John Gruber）。它允许人们
“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML（或者HTML）文档”。这种语言吸
收了很多在电子邮件中已有的纯文本标记的特性。 –wiki百科

  Markdown这种语法，应该称为语言，我在以前就使用过了，不过那时候用的比较简单，工具是用的网易云笔记，主要作用呢，就是报告一下进度，以及目前的问题之类的，但是那个时候，无非就是天天写流水一般的总结，虽然相比其他人来说，我的流水算是没那么单纯，但是还是只用到了几个字符而已，”#”控制标题大小，”···”后跟随代码……等等一系列，简单，易用，写出来的东西耶尚可，就是有一点，感觉整个编辑流程，就像在word中用空格键去代替的首行缩进那样，那种不专业感，看似没事，但是随着打开的工具不同，会出现显示错误等一系列的问题，最可怕的是有时候会显得自己写出来的东西很含糊。

  为了贯彻写博客这件事，我这几天读了两篇关于Markdown的文章，看了许久也没有去看各种字符的作用，一直在了解着Markdown的背景和发展，感觉最明显的就是：对于Markdown的存在，如果是一个日常经常使用Word做报告或者写作的人来说，这就是一种没什么必要，且多余的东西。想想，一个人用了大半年去学习一系列的办公软件（对于非it从业者来说，或是一种常见的现象），基本上日常都是使用这一工具去工作，特别一些文职人员，对于他们来说，其实再学习Markdown，无疑是累赘而且多余的。但是反观我这样的码农，每天学习着不同的框架，踩着不一样的坑，本来的工作就是多变，每天就经历着不同的学以致用，再多学一种语法，对日常丝毫没有影响。或许有一天会衍生出一种用Markdown标记的编程语言，因为这种语法，看着那些跟注释一样的标记，着实让我感觉到很舒服，大致看过一些字符的意义就能一眼看出各处的意思，好比在接手一份老代码的时候，打开了一篇写满注释的代码，久旱逢甘霖一般。

  好吧，我上面两段话对于使用Markdown毫无帮助，都是废话，说实在我现在也不适合说什么帮助一类的东西，我也刚开始学习，但是这里推荐一下sainho这位作者发布在少数派上的两篇文章：《Markdown 完全入门(上篇/下篇)》文章通篇对于Markdown的语法特性，以及场景，和Word对比概述的非常完整，虽然对于入手帮助不大，但是基于搞懂Markdown这个问题，基本上是很完善了，作者也在上篇中明确说明了，Markdown这个东西不能仅靠三言两语来解释的。就好比经典的《XXX从入门到___》基本上是不可能的，首先书籍就是千页起步，一篇文章下来不超过1W字，对于一名读者来说，面对屏幕过长时间的阅读会产生疲劳，更多的人也在读书的过程中逐渐放弃，所以我觉得，学习的一种工具过程，应该是去了解它，知道它的历史，以及搞明白什么时候使用到它，然后知道怎么去找到使用方法，就是最佳的学习了，剩下的，就是在使用期间，通过学到的方法需寻找解决方法（其实我认为就是关键字，有时候寻找篇文章或者寻找一个东西，最大的问题就是关键字），这样就可以完美的解决问题，把坑一个个踏过，如果上面作者所说的，Markdown依赖的不是视觉化的工具，而是肌肉记忆，最后实现编辑内容和排版一气呵成。
